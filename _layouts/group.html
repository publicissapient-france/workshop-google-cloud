Les variables: projectName, projectId, cloudSdkName

 

Votre project Compute Engine s’appelle {{page.projectName}} et son project ID est {{page.projectId}}

Prérequis

Installer le “Google Cloud SDK for Java”

Pour le télécharger aller sur cette page : https://developers.google.com/cloud/sdk/

Dézipper dans le répertoire de votre choix.

Ajouter dans votre PATH l’accès aux binaires:

export PATH=$PATH:<YOUR PATH TO SDK>/{{page.cloudSdkName}}/bin

S’authentifier avec votre project {{page.projectName}} ({{page.projectId}})

La première chose avant de commencer est de vous authentifier et définir votre projet comme celui à utiliser par défaut lors des prochaines utilisations de la ligne de commande.

Pour cela taper la commande suivante (il vous sera demandé de fournir le nom de votre projet, taper : “{{page.projectName}}” cela vous évitera d’avoir à rappeler le nom du projet à chaque utilisation de la ligne de commande) :

gauth --login

gcutil ssh demo-instance (creation de clef ssh, prend 5 minutes)

Créer une instance compute engine

Aller sur la console de votre projet:

https://cloud.google.com/console#/project/apps~{{page.projectId}}

Connexion en ssh à l’instance

gcutil ssh demo-instance

Installation de Tomcat7

sudo apt-get install tomcat7

sudo gsutil cp gs://petclinic/petclinic.war /var/lib/tomcat7/webapps/

gcutil adddisk --source_snapshot=demo-snapshot demo-instance-2-disk

gcutil addinstance demo-instance-2 --disk demo-instance-2-disk
